Success metrics:
- Every RFQ pack has a hash recorded.
- Delta triggers exactly once per new version.
- Tender cannot move to “submitted” unless delta gate is cleared (soft gate in MVP; hard gate optional).

Constraints:
- Non-breaking; additive tables/columns only.
- Idempotent jobs.

Checklist:
- [ ] Add tender_rfq_versions table (document_id, tender_id, sha256, created_at)
- [ ] Add tenders.delta_status + last_rfq_hash columns
- [ ] Update document upload flow to compute hash (server-side) or store for background job
- [ ] Add UI banner on tender detail when delta pending

Validation:
- Upload RFQ v1 → no delta.
- Upload RFQ v2 (different file) → delta flagged + notification + optional tasks created.
- Clear delta → allow submit.
